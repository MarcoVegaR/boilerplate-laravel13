La auditoría unifica eventos de modelos y eventos de seguridad para facilitar revisiones operativas.

## Filtros más útiles

- **Fuente** para separar seguridad de cambios de modelos.
- **Actor** para seguir la actividad de una persona concreta.
- **Evento** para revisar solo creaciones, actualizaciones o eventos de autenticación.
- **Rango de fechas** para limitar la investigación.

## Flujo recomendado de investigación

1. Define primero el rango temporal.
2. Reduce por fuente si ya sabes si buscas seguridad o cambios de datos.
3. Filtra por actor o evento para disminuir ruido.
4. Abre el detalle del registro cuando necesites contexto adicional.

## Consejo

Si el resultado es muy amplio, no sigas desplazándote sin criterio: ajusta filtros y vuelve a ejecutar la consulta para dejar un rastro más claro de la revisión.
